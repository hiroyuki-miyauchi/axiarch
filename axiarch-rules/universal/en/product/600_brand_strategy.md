## Part I. Brand Philosophy

### 1.1. What a Brand Is

- **Rule 14.001**: Trivializing brand as "logo, color, and font" is prohibited. A brand is **"the totality of emotional associations a user holds"**
- **Simon Sinek's Golden Circle**:

```
Why (Why do we exist?) ← The core of brand
  ↓
How (How do we deliver it?)
  ↓
What (What do we provide?) ← Where most companies start

Rule: All brand communication must start from "Why"
```

### 1.2. Brand as a Moat

- **Rule 14.002**: A strong brand generates the following economic value
  - **Lower CAC**: Higher awareness increases inbound, lowering CAC
  - **Price premium**: Justify higher prices for equivalent features (Apple vs. commodity PC)
  - **Recruiting advantage**: Attracts high-quality talent
  - **Partnership magnetism**: Trusted brands attract partners

### 1.3. Brand Promise

- **Rule 14.003**: Define a **Brand Promise** at the core of the brand
  - Brand Promise expresses in one sentence "what users can reliably expect every time"
  - Brand Promise is defined by the CEO / CPO, not the marketing department
  - Examples: "Amazon: The most customer-centric company on Earth" / "Apple: Innovative technology for everyone"

### 1.4. Brand Redefined for the AI Era

- **Rule 14.004**: In the AI era, brands are "experienced" across 3 layers. Ensure consistency across all layers

```
┌──────────────────────────────────────────┐
│  Layer 1: Human-Delivered Brand          │
│  → Sales, Support, Events, Employee acts │
├──────────────────────────────────────────┤
│  Layer 2: Product-Delivered Brand        │
│  → UI/UX, Microcopy, Notifications      │
├──────────────────────────────────────────┤
│  Layer 3: AI-Delivered Brand             │
│  → Chatbot, AI Copilot, Voice Assistant  │
│  → AI Agent auto-responses & recs        │
└──────────────────────────────────────────┘

Rule: Layer 3 must NEVER contradict Layers 1 & 2.
If a single AI utterance betrays the Brand Promise,
it is not a "bug"—it is a "brand damage incident."
```

- **Anti-Pattern**: Telling the AI chatbot to "just be friendly" without referencing the Brand Voice definition → brand collapse

### 1.5. Brand Economic Value Framework

- **Rule 14.005**: Report brand asset value to the executive team using the following framework

| Method | Description | Application |
|:-------|:-----------|:-----------|
| **Brand Strength Index** | 4-stage score: Awareness → Consideration → Preference → Loyalty | Quarterly review |
| **Brand Contribution** | Estimated % of revenue attributable to brand | Annual planning |
| **Brand-CAC Attribution** | Organic inflow ÷ Total inflow = Brand contribution rate | Marketing budget allocation |
| **Price Premium Analysis** | Price gap vs. comparable competitor × sales volume | Pricing strategy review |

- **Prohibition**: Treating brand investment as an "expense" and cutting it without ROI measurement is prohibited. Brand is a depreciating intangible asset requiring continuous investment

### 1.6. Brand Time Horizon Design

- **Rule 14.006**: Manage brand investment across 3 time horizons to prevent short-term bias

```
Horizon          Allocation    Purpose
─────────────────────────────────────────
Short (0-6 mo)     30%        Campaign: Awareness & lead generation
Mid (6-24 mo)      40%        Positioning: Category differentiation
Long (2-5 yr)      30%        Equity: Trust accumulation as brand asset
```

- **Law**: Evaluating brand investment solely by short-term campaign metrics (CPA, ROAS) is prohibited
- **Mandate**: Annual brand budget reviews must explicitly show allocation across all 3 horizons and obtain executive approval

### 1.7. Brand Flywheel

- **Rule 14.007**: Design brand as a **self-reinforcing loop (Flywheel)**, not a "campaign"

```
Brand Flywheel:
┌─────────────────────────────────────────┐
│                                         │
│  Brand Trust → Lower Acquisition Cost   │
│       ↑                    ↓            │
│  Brand Reinforcement ← Growth ← Organic│
│                                         │
└─────────────────────────────────────────┘

1. Trust: Consistent brand experience accumulates trust
2. Acquisition: Trust increases organic traffic, lowering CAC
3. Growth: Low CAC generates reinvestable growth capital
4. Reinforcement: Growth expands brand awareness, further accumulating trust
```

- **Law**: Set KPIs for each Flywheel stage and identify bottlenecks quarterly
- **Anti-Pattern**: Treating brand as "one-way broadcasting" without loop design → inability to escape ad spend dependency

## Part II. Brand Identity Design

### 2.1. Brand Pyramid

- **Rule 14.010**: Define brand identity in the following 5-layer structure

```
    ┌─────────────────┐
    │   Brand Essence  │ (Why: Reason for existence)
    ├─────────────────┤
    │ Brand Personality│ (Character / Persona)
    ├─────────────────┤
    │  Brand Values    │ (Values that constrain behavior)
    ├─────────────────┤
    │  Brand Promise   │ (Promise to users)
    ├─────────────────┤
    │  Brand Position  │ (Market positioning)
    └─────────────────┘
```

### 2.2. Brand Personality

- **Rule 14.011**: Define brand personality through up to 5 adjectives framed as "if the brand were a person"
  - Example (Stripe): Polished / Engineer-founded / Straightforward / Meticulous / Innovative
  - Example (Notion): Playful / Minimalist / Inclusive / Thoughtful

### 2.3. Brand Values

- **Rule 14.012**: Brand values also define "what we don't do"

```yaml
brand_values_template:
  values:
    - name: "Radical Simplicity"
      do: "Make complex things appear simple"
      dont: "Add features that complicate the UI"
    - name: "Transparent by Default"
      do: "Communicate errors honestly and humanly"
      dont: "Show meaningless error messages like 'An error occurred'"
```

### 2.4. Mission / Vision / Values Alignment

- **Rule 14.013**: Verify that Mission, Vision, and Values are consistent and mutually reinforcing

| Element | Definition | Time Horizon |
|:--------|:-----------|:------------|
| **Mission** | Why we exist / current purpose | Present progressive |
| **Vision** | The future we're building toward | 5–10 years |
| **Values** | How we act | Immutable |

### 2.5. Brand Archetype

- **Rule 14.014**: Define brand character using the **12 Archetypes framework** as the foundation for consistent storytelling

| Category | Archetype | Keywords | Example |
|:---------|:---------|:---------|:--------|
| **Stability** | Caregiver | Compassion, Protection | Johnson & Johnson |
| **Stability** | Ruler | Control, Leadership | Mercedes-Benz |
| **Independence** | Explorer | Freedom, Discovery | Patagonia |
| **Independence** | Sage | Wisdom, Truth | Google |
| **Mastery** | Hero | Courage, Achievement | Nike |
| **Mastery** | Magician | Transformation, Vision | Apple |
| **Belonging** | Everyman | Familiarity, Empathy | IKEA |
| **Belonging** | Jester | Humor, Fun | M&M's |

- **Law**: The chosen Archetype must be documented in the Brand Playbook and used as the consistent "personality axis" across all content and campaigns
- **Anti-Pattern**: Changing Archetype per campaign → brand "multiple personality disorder"

### 2.6. Brand Story Framework

- **Rule 14.015**: Manage brand stories in a 3-layer structure, deploying the appropriate layer per context

```
┌──────────────────────────────────────┐
│  Origin Story                         │
│  → Founder's original experience      │
│  → Usage: About Us, Recruiting, IR    │
├──────────────────────────────────────┤
│  Customer Story                       │
│  → User transformation narratives     │
│  → Usage: LP, Case Study, Social      │
├──────────────────────────────────────┤
│  Future Story                         │
│  → The future the brand will create   │
│  → Usage: Vision, Roadmap             │
└──────────────────────────────────────┘
```

- **Mandate**: All 3 story layers must be consistent with the Brand Promise. Conduct annual alignment checks

### 2.7. Competitive Positioning Matrix

- **Rule 14.016**: Visualize differentiation from key competitors using a **Perceptual Map**, updated semi-annually

```
Axis selection criteria:
1. Choose the 2 axes most influential in user purchase decisions
2. Prioritize axes that reflect the brand's unique strengths

Example:
  Y-axis: Intuitiveness of use (High ↑ Low)
  X-axis: Price range (Low ← → High)

  Our brand ★ captures the "High intuitiveness × Fair price" position
```

- **Law**: Repositioning is a strategic decision requiring CEO approval; marketing teams must not execute it unilaterally

## Part III. Visual Identity Management

### 3.1. Visual System Components

- **Rule 14.020**: Compose visual identity from the following elements

| Element | Managed File | Update Frequency |
|:--------|:-----------|:----------------|
| **Logo** | SVG (color, white, black, mark only) | Brand revision only |
| **Color Palette** | Primary / Secondary / Accent | Annual review |
| **Typography** | Heading / Body / Code font definitions | Annual review |
| **Icon System** | Unified icon library | When features are added |
| **Imagery Style** | Photo / illustration / screenshot tone | Annual review |
| **Motion** | Micro-animation principles | Annual review |

### 3.2. Logo Usage Guidelines (Brand Safety)

- **Rule 14.021**: The following rules are absolute for logo usage

```
Permitted:
✅ Use official SVG files without modification
✅ Maintain clear space ≥ 0.5× logo height on all sides

Prohibited:
❌ Changing aspect ratio / distorting scale
❌ Overlaying shadows, gradients, or textures on the logo
❌ Using unauthorized color variations
❌ Insufficient contrast with background (below WCAG AA)
```

### 3.3. Design Token Hierarchy Management

- **Rule 14.022**: Manage all design elements as Design Tokens and establish a single source of truth between code and Figma

```json
{
  "color": {
    "brand": {
      "primary":   { "value": "#6366F1", "description": "Indigo: Main brand color" },
      "secondary": { "value": "#8B5CF6", "description": "Violet: Accent color" }
    },
    "semantic": {
      "success": { "value": "{color.brand.primary}" },
      "error":   { "value": "#EF4444" }
    }
  }
}
```

### 3.4. Design Token Architecture 2.0

- **Rule 14.023**: Design Tokens must conform to the **DTCG standard** and be managed in a 3-layer architecture

```
┌──────────────────────────────────────────┐
│  Layer 1: Global Tokens (raw values)      │
│  → color.blue.500: "#3B82F6"             │
│  → spacing.4: "16px"                     │
├──────────────────────────────────────────┤
│  Layer 2: Semantic Tokens (meaning)       │
│  → color.brand.primary: "{color.blue.500}"│
│  → color.feedback.error: "{color.red.500}"│
├──────────────────────────────────────────┤
│  Layer 3: Component Tokens (component)    │
│  → button.primary.bg: "{color.brand.primary}"│
│  → button.primary.text: "{color.white}"   │
└──────────────────────────────────────────┘
```

- **Law**: Component developers must reference Layer 3 only. Direct reference to Layer 1 is prohibited as "token bypass"
- **SSOT**: The source of truth for tokens is `tokens.json`, auto-synced to Figma Variables and CSS Custom Properties

### 3.5. Multi-Theme & Dark Mode Protocol

- **Rule 14.024**: Support the following theme variations as standard

| Theme | Requirements | Priority |
|:------|:-----------|:---------|
| **Light Mode** | Default theme | 🔴 Required |
| **Dark Mode** | Implemented via Semantic Token value switching | 🔴 Required |
| **High Contrast** | WCAG AAA compliant contrast ratios | 🟠 Recommended |
| **Reduced Motion** | `prefers-reduced-motion` support | 🔴 Required |

- **Law**: Dark Mode is NOT "color inversion." Background, text, border, shadow, and image brightness must each be controlled individually at the Semantic Token layer
- **Prohibition**: Hardcoded color codes (`#ffffff`, `rgb(0,0,0)`) in UI are prohibited. All colors must be specified via tokens

### 3.6. Accessibility-First Visual Design

- **Rule 14.025**: Brand visuals must meet **WCAG 2.2 Level AA** as a minimum

| Requirement | Standard | Verification Tool |
|:-----------|:--------|:-----------------|
| **Text contrast** | Normal ≥ 4.5:1 / Large ≥ 3:1 | axe / Lighthouse |
| **Non-text contrast** | UI elements ≥ 3:1 | Colour Contrast Analyser |
| **Focus indicator** | Visible focus ring on all interactive elements | Manual testing |
| **Text size** | Body ≥ 16px (rem units recommended) | CSS Audit |
| **Not color-only** | Error states must use icons/text in addition to color | Design review |

### 3.7. Motion Design Language

- **Rule 14.026**: Define brand motion across 3 axes to establish a unified movement "personality"

```yaml
motion_design_language:
  duration:
    instant: "100ms"   # Feedback (button press)
    fast: "200ms"      # Small state changes (toggle, expand)
    normal: "300ms"    # Medium transitions (modal, drawer)
    slow: "500ms"      # Large transitions (page, loading complete)
  easing:
    standard: "cubic-bezier(0.2, 0, 0, 1)"
    enter: "cubic-bezier(0, 0, 0, 1)"
    exit: "cubic-bezier(0.2, 0, 1, 1)"
  purpose:
    feedback: "Instant response to user action"
    orientation: "Helps understand spatial relationships"
    delight: "Subtle expressions of brand personality"
```

- **Law**: When `prefers-reduced-motion: reduce` is detected, immediately disable `delight` category animations. `feedback` and `orientation` must never be omitted

### 3.8. Figma-Code SSOT Protocol

- **Rule 14.027**: Establish the following unidirectional sync pipeline to prevent design-code drift

```
Figma Variables (design source of truth)
    ↓ [Auto-export: Tokens Studio / Style Dictionary]
tokens.json (DTCG format)
    ↓ [Build pipeline: Style Dictionary / Token Transformer]
CSS Custom Properties / Tailwind Config / etc.
    ↓ [Runtime]
Product UI
```

- **Law**: Designer changes Figma → token JSON auto-updates → code reflects changes. This unidirectional flow must be maintained
- **Prohibition**: Developers adding color values independently in CSS ("shadow tokens") is prohibited. All values must originate from Figma Variables
- **Audit**: Quarterly scan of all CSS color/font values to verify 0 off-token values

## Part IV. Voice, Tone & Copywriting

### 4.1. Brand Voice

- **Rule 14.030**: Define Brand Voice as **"a permanent character that never changes"** and distinguish it from Tone (situational adjustment)

```
Brand Voice (Immutable):
- Direct; no wasted words
- Technically accurate but never alienating through jargon
- Human, with a sense of humor
- Confident but never arrogant

Brand Tone (Situational):
- During errors: More empathetic, support-focused
- New feature launch: More excited, energetic
- Terms of service: More precise, formal
```

### 4.2. Copywriting Principles

- **Rule 14.031**: Apply the following 5 principles to all external-facing copy

| Principle | Rule |
|:----------|:----|
| **Clarity First** | Clarity over cleverness. Is it immediately understandable on first read? |
| **Benefits over Features** | Write the benefit the user gets, not the feature itself |
| **Active Voice** | Use active voice over passive voice |
| **No Jargon** | Don't use industry buzz words or internal terminology |
| **Read Aloud Test** | Verify it doesn't sound unnatural when read aloud |

### 4.3. Button Copy Standards

- **Rule 14.032**: CTA button copy must be designed with these rules

```
Good button copy:
✅ Start with a verb: "Get started," "Try free," "Invite your team"
✅ Be specific: "Send feedback" instead of "Submit"
✅ User perspective: Self-referential form like "Start my [X]"

Bad button copy:
❌ "Yes," "OK," "Submit" (unclear what happens)
❌ "Click here" (describes the action, not the value)
❌ ALL CAPS everywhere (feels aggressive)
```

### 4.4. Error Message Design Protocol

- **Rule 14.033**: Error messages are the moment brand voice is most tested. Design them with this structure

```
3 Components of an Error Message:
1. What happened (What): Communicate the state clearly
2. Why it happened (Why): Provide the cause when possible
3. What to do next (How): Show a clear next action

Good example:
"This feature is available on the Pro plan.
 You're currently on the Starter plan.
 [Upgrade to Pro] to start using this feature."

Bad example:
"Error occurred (Error: 403)"
```

### 4.5. Multi-Context Tone Matrix

- **Rule 14.034**: Brand Voice is immutable, but Tone is designed across a **"Situation × Channel × Emotion"** 3-axis matrix

| Situation | In-App | Email | Social |
|:----------|:-------|:------|:-------|
| **Success** | Celebratory, simple | Detailed, next steps | Shareable, visual |
| **Error** | Empathetic, immediate fix | Technical, detailed | Swift, transparent |
| **New Feature** | In-product guide | Benefit-focused | Teaser, anticipation |
| **Billing** | Clear, reassuring | Formal, audit-ready | Not used |
| **Security Alert** | Urgent, action-oriented | Urgent, step-by-step | Brief, official link |

### 4.6. Inclusive Language Protocol

- **Rule 14.035**: All official communications must adhere to inclusive language guidelines

| Category | Avoid | Prefer |
|:---------|:------|:-------|
| **Gender** | "he/she," gendered terms | "they," gender-neutral terms |
| **Disability** | "disabled person" | "person with a disability" |
| **Tech level** | "It's easy," "Anyone can do it" | "Follow these steps to set up" |
| **Age** | "For elderly users" | "For users of all ages" |
| **Culture** | Culture-specific metaphors | Universal expressions |

### 4.7. Microcopy & UX Writing Standards

- **Rule 14.036**: Standardize in-product microcopy by UI element category

| UI Element | Principle | Example |
|:----------|:---------|:--------|
| **Empty State** | Show next action | "No posts yet. Create your first post" |
| **Loading** | Communicate progress | "Loading data..." / "3/5 complete" |
| **Tooltip** | 1 sentence, start with verb | "Click to edit your profile" |
| **Confirmation** | State the consequence | "This cannot be undone. Delete?" |
| **Success** | Give achievement feeling | "Saved ✓" |

### 4.8. Multi-Language Copy Governance

- **Rule 14.037**: Manage multi-language copy quality with the following criteria

| Criteria | Translation | Transcreation |
|:---------|:-----------|:-------------|
| **Scope** | Technical docs, help articles, legal | Taglines, ad copy, CTAs |
| **Goal** | Accurate meaning transfer | Emotional impact reproduction |
| **Owner** | Translator | Copywriter + native reviewer |
| **QA** | Terminology consistency, grammar | Brand tone fit, cultural adaptation |

- **Law**: Taglines and Brand Promise must go through Transcreation. Direct translation is prohibited
- **Mandate**: Maintain a glossary per language to ensure translator consistency

## Part V. AI-Native Brand Governance

> **Supreme Directive**: In an era where AI "acts on behalf of" the brand, all AI utterances and generated content are official brand expressions. AI output quality management is a core brand governance responsibility.

### 5.1. AI-Generated Content Brand Compliance Gate

- **Rule 14.070**: AI-generated content (text, image, video) must pass the following **Brand Compliance Scorecard** before publication

| Check Item | Standard | Pass Threshold |
|:----------|:---------|:-------------|
| **Voice Match** | Alignment with Brand Voice 5 attributes | All attributes matched |
| **Tone Accuracy** | Alignment with Tone Matrix for context | Matrix compliant |
| **Factual Accuracy** | Accuracy of facts and figures | 0 errors |
| **Inclusive Language** | §4.6 protocol compliance | 0 violations |
| **Visual Consistency** | Brand color/typography compliance | Token compliant |

### 5.2. LLM Brand Persona Protocol

- **Rule 14.071**: Define AI Chatbot / Copilot persona with this structure

```yaml
llm_brand_persona:
  name: "[Brand Name] AI Assistant"
  personality: "Per §2.2 Brand Personality"
  voice: "Per §4.1 Brand Voice"
  constraints:
    - Do not criticize competitors
    - Do not mention unreleased features/roadmap
    - Do not provide medical/legal/financial advice
    - Do not request personal information
    - Do not express political/religious positions
  escalation:
    trigger: "Strong negative user emotion / safety-related inquiry"
    action: "Immediate escalation to human support"
```

- **Prohibition**: Making the AI pretend to be human is strictly prohibited. When asked, AI must honestly disclose its nature

### 5.3. AI-Generated Visual Brand Compliance

- **Rule 14.072**: When using AI image generation for brand assets, the following guardrails apply

| Guardrail | Requirement |
|:----------|:-----------|
| **Brand Color Enforcement** | Explicitly specify brand palette in prompts |
| **Style Consistency** | Use prompt templates conforming to Imagery Style guide |
| **Human Representation** | Prohibit generating images resembling real individuals |
| **Copyright Risk** | Negative prompts to avoid imitating existing artwork |
| **Metadata** | Record AI-generated status in EXIF / C2PA |

### 5.4. Synthetic Media & Deepfake Defense

- **Rule 14.073**: Implement these defenses against Deepfake abuse of the brand

| Defense | Action | Frequency |
|:--------|:------|:---------|
| **Monitoring** | Social monitoring for brand name + deepfake keywords | Daily |
| **Official Channels** | Publish list of official SNS accounts & websites | Always |
| **C2PA Signing** | Attach Content Credentials to official content | On publish |
| **Legal Templates** | Prepare DMCA / platform report templates | Pre-prepared |

### 5.5. AI Agent Brand Delegation Protocol

- **Rule 14.074**: When AI agents communicate externally on behalf of the brand, the following permission matrix applies

| Action | AI Permission | Human Approval |
|:-------|:------------|:-------------|
| FAQ / Help responses | ✅ Autonomous | Not required |
| Refund / compensation proposals | 🟠 Propose only | Required |
| Official statements | ❌ Prohibited | Human creates & publishes |
| Social media posts | 🟠 Draft only | Required |
| Contractually binding responses | ❌ Prohibited | Human handles |

- **Kill Switch**: An emergency stop mechanism that can halt all AI external communications within 30 seconds must be implemented

### 5.6. Golden Dataset Protocol

- **Rule 14.075**: Manage and operate the following **Golden Dataset** to ensure AI output brand quality

| Dataset | Content | Update Frequency |
|:--------|:-------|:-----------------|
| **Voice Exemplars** | 50+ approved copies embodying Brand Voice | Quarterly |
| **Tone Samples** | Model response sets per situation (§4.5 Tone Matrix) | Quarterly |
| **Visual References** | Brand-compliant image/design reference set | Semi-annual |
| **Anti-Examples** | Specific examples of brand violations (what’s NG) | As needed |

- **Law**: Only content approved by Brand Owner may be stored in the Golden Dataset. Unapproved content contamination is treated as "data poisoning" requiring immediate remediation
- **SSOT**: The source of truth for the Golden Dataset is stored in a version-controlled repository with unified reference paths from AI systems

### 5.7. AI Brand Drift Detection

- **Rule 14.076**: Quantitatively detect "drift" where AI output deviates from brand standards

| Detection Method | Content | Threshold |
|:----------------|:-------|:----------|
| **Voice Consistency Score** | Similarity score with Golden Dataset (embedding distance) | Score < 0.85 triggers alert |
| **Tone Drift Monitor** | Deviation rate from situational Tone Matrix | Deviation > 15% triggers auto-escalation |
| **Hallucination Rate** | Rate of factual misrepresentation about the brand | ≥ 0.5% triggers immediate fix |
| **Sentiment Deviation** | Trend in user sentiment toward AI responses | Negative rate > 150% vs. prior month |

- **Automation**: Drift detection runs as daily batch; threshold breaches auto-notify Brand Guardian
- **Remediation**: On drift detection: ① Prompt adjustment, ② Golden Dataset update, ③ Model fine-tuning, in that order

### 5.8. Multi-Agent Brand Orchestration

- **Rule 14.077**: When multiple AI agents simultaneously act on behalf of the brand, apply the following orchestration protocol

```
┌──────────────────────────────────────────┐
│  Brand Orchestration Layer               │
│  ┌────────────────────────────────────┐  │
│  │ Central Brand Config (SSOT)        │  │
│  │ → Brand Voice definition            │  │
│  │ → Tone Matrix                      │  │
│  │ → Prohibited actions list            │  │
│  │ → Golden Dataset reference path     │  │
│  └────────────┬───────────────────────┘  │
│               ↓                          │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐   │
│  │Agent │ │Agent │ │Agent │ │Agent │   │
│  │Chat  │ │Email │ │Social│ │Voice │   │
│  └──────┘ └──────┘ └──────┘ └──────┘   │
└──────────────────────────────────────────┘
```

- **Law**: Each agent must NOT maintain its own Brand Voice definition. All must uniformly read from Central Brand Config (single configuration source)
- **Anti-Pattern**: Defining Voice/Tone in individual prompts per agent → "personality inconsistency" between agents
- **Sync Protocol**: Central Brand Config updates must propagate to all agents within 15 minutes

## Part VI. Brand Experience (BX) Design

### 6.1. Brand Experience Touchpoint Map

- **Rule 14.040**: Map all touchpoints between users and the brand and design the brand experience at each one

```
Touchpoint Examples:
Awareness: Ads, PR, social posts, word of mouth
Consideration: LP, demo videos, comparison sites
Experience: Signup, Onboarding, first use
Retention: Email notifications, live chat, in-product experience
```

- **Law**: Web3 initiatives must "strengthen" brand credibility. Use for speculation or FOMO incitement violates §11.3 ethical standards

## Part XVIII. Share of Model & AI Discoverability

> **Supreme Directive**: In the AI era, "how LLMs perceive a brand" is equally or more important than search engine rankings. Share of Model (brand share within AI models) is a new brand strategy KPI.

### 18.1. Share of Model Audit Framework (SoM)

- **Rule 14.170**: Quantitatively audit brand recognition across major LLMs

```
Share of Model (SoM) Audit Process:
1. Define prompt test set (50+ questions)
   - Direct brand query: "What is [brand]?"
   - Category query: "Best [category] recommendation?"
   - Comparison query: "[brand] vs [competitor]"
2. Execute across major LLMs (GPT-4o, Gemini, Claude, Llama, Mistral)
3. Score response accuracy, citation rate, and sentiment
4. Report monthly to Brand Owner
```

- **KPI**: SoM Score = (Accurate citations / Total tests) × 100
- **Target**: SoM Score ≥ 70% (category leader benchmark)

### 18.2. Machine-Readable Brand Assets

- **Rule 14.171**: Maintain machine-readable assets for AI agents to correctly understand the brand

| Asset | Content | Format |
|:-----|:-------|:-------|
| **LLMs.txt** | Brand information summary for AI engines | Text |
| **Brand Schema** | Schema.org-compliant structured brand data | JSON-LD |
| **Brand API** | Programmatic brand information delivery | REST/GraphQL |
| **Brand Knowledge Base** | Structured FAQ and glossary data | Markdown + JSON |

### 18.3. AI Agent Commerce Readiness

- **Rule 14.172**: Preparation for the era when AI agents (A2A / MCP) autonomously recommend and purchase brand products

| Preparation Item | Content |
|:----------------|:-------|
| **Machine-Readable Pricing** | Publication of structured pricing data |
| **Product Catalog API** | Product catalog accessible by AI agents |
| **Agent-Facing Brand Brief** | Concise brand guidelines for AI agents |
| **Trust Signal** | Digital certification demonstrating credibility (C2PA, Verified Org, etc.) |

### 18.4. Brand Narrative Seeding for LLMs

- **Rule 14.173**: Strategy to infuse accurate brand narrative into LLM training data

| Initiative | Method |
|:----------|:------|
| **Authority Source Publishing** | Consistent brand information in authoritative media |
| **Knowledge Graph Management** | Accuracy management of Wikipedia / Wikidata / Crunchbase |
| **Structured Content** | Answer-First format official content (§13.1 GEO coordination) |
| **Citation Worthiness** | Regular publication of original data and proprietary research |

- **Cross-Reference**: Coordinates with §13 GEO & E-E-A-T Brand Trust Strategy

## Part XIX. Data-Driven Brand Intelligence 2.0

> **Supreme Directive**: Traditional quarterly survey-based brand measurement is "too slow." Transition to "always-on" brand intelligence leveraging AI and real-time data, optimizing brand at the same velocity as product.

### 19.1. Continuous Brand Tracking

- **Rule 14.180**: Transition from quarterly surveys to "always-on" brand tracking

| Method | Content | Frequency |
|:------|:-------|:---------|
| **Always-On Survey** | Always-on micro-surveys (3 questions or fewer) | Daily |
| **Social Listening AI** | Real-time sentiment tracking via NLP analysis | Real-time |
| **Search Signal** | Brand search volume and trend analysis | Daily |
| **Product Signal** | Brand estimation from in-product behavioral data | Real-time |

### 19.2. 3-Layer Brand Metrics Architecture

- **Rule 14.181**: Manage brand metrics in the following 3-layer architecture

```
3-Layer Brand Metrics:
┌──────────────────────────────────────────┐
│  Layer 1: Behavioral                     │
│  → Engagement rate / Churn rate          │
│  → Usage frequency / Feature adoption    │
├──────────────────────────────────────────┤
│  Layer 2: Perceptual                     │
│  → Brand trust score / Sentiment score   │
│  → Share of Voice / Share of Model       │
├──────────────────────────────────────────┤
│  Layer 3: Financial                      │
│  → Price premium / CLV / CAC reduction   │
│  → Brand-attributed organic traffic %    │
└──────────────────────────────────────────┘
```

- **Law**: Calculate an integrated Brand Equity Index across all 3 layers monthly and report as executive KPI

### 19.3. Brandformance Framework

- **Rule 14.182**: Unified measurement framework for branding and performance marketing

| Principle | Content |
|:---------|:-------|
| **Unified Attribution** | Track all campaigns on both "awareness" and "action" axes |
| **Brand Lift + Conversion** | Visualize brand lift and conversion on a single dashboard |
| **Long-term Incrementality** | Calculate composite ROI from short-term ROAS + long-term brand lift |
| **Cross-Channel Integration** | Quantify brand contribution across channels |

- **Anti-Pattern**: Leaving branding budget as "unmeasurable" → inability to justify brand investment

### 19.4. Competitive Brand Intelligence

- **Rule 14.183**: Real-time monitoring of competitive brands with automated alerts

| Monitoring Item | Method | Alert Threshold |
|:---------------|:------|:---------------|
| **Competitor positioning change** | Track messaging and pricing changes | Upon detecting major changes |
| **Competitor SoV fluctuation** | Weekly Share of Voice comparison | ±20% or greater change |
| **Competitor brand campaigns** | Monitor ad spend and PR activity | Large campaign detection |
| **Review/reputation changes** | Competitor product review score changes | ±0.5pt or greater change |

### 19.5. Predictive Brand Analytics

- **Rule 14.184**: Leading indicator management for brand health via predictive analytics

| Leading Indicator | Prediction Target | Method |
|:----------------|:-----------------|:------|
| **Sentiment trend slope** | Brand sentiment change over next 30 days | Time series analysis |
| **Search trends** | Brand awareness fluctuation prediction | Google Trends + internal data |
| **Engagement decline** | Early churn risk detection | Behavioral pattern anomaly detection |

- **Law**: When leading indicators show negative trends, present root cause analysis and countermeasure proposals to Brand Owner within 14 days

## Part XX. Brand Compliance Automation & CI/CD

> **Supreme Directive**: Relying solely on human review for brand compliance has limitations in both scalability and speed. Integrate brand governance into the CI/CD pipeline and automate it.

### 20.1. Brand Linting Pipeline

- **Rule 14.190**: Integrate the following brand compliance checks into the CI/CD pipeline

| Check Item | Method | Gate |
|:----------|:------|:-----|
| **Color Token Validation** | Detect non-token color values from CSS | 0 findings = Pass |
| **Typography Check** | Detect unapproved font usage | 0 findings = Pass |
| **Copy Voice Score** | AI-analyzed Brand Voice compliance score | ≥ 0.85 = Pass |
| **Alt Text Compliance** | Image alt attribute presence and quality check | 100% = Pass |
| **Inclusive Language** | Automated term check per §4.6 | 0 violations = Pass |

- **Law**: Brand Linting failures BLOCK the CI pipeline (cannot be treated as Warnings)

### 20.2. Design System Governance Automation

- **Rule 14.191**: Automate design system change management

| Automation Area | Content |
|:--------------|:-------|
| **Token Change Impact Analysis** | Auto-analyze scope of impact from token changes |
| **Visual Regression Testing** | Screenshot comparison for UI change detection |
| **Component Usage Audit** | Auto-detect usage of deprecated components |
| **Figma-Code Drift Detection** | Auto-detect §3.8 SSOT Protocol violations |

### 20.3. AI-Powered Brand Review Bot

- **Rule 14.192**: Implement a bot for automated brand compliance checking during PR reviews

```
Brand Review Bot Scope:
1. Voice score for microcopy / error messages
2. Brand color compliance on UI screenshots
3. Style guide conformity for new icons / images
4. Accessibility standard (§3.6) auto-verification
5. Inclusive Language (§4.6) auto-scan
```

- **Mandate**: Brand Review Bot findings are mandatory PR review items; merging with unresolved findings is prohibited

### 20.4. Brand Compliance SLA

- **Rule 14.193**: Define remediation SLAs for brand compliance violations

| Severity | Definition | Remediation Deadline |
|:--------|:----------|:-------------------|
| **Critical** | Brand Identity damage (logo alteration, color deviation) | Within 24 hours |
| **Major** | Voice/Tone deviation, token violation | Within 1 sprint |
| **Minor** | Minor copy correction, spacing adjustment | Next release |

- **Escalation**: When 2 or more Critical violations occur simultaneously, Brand Guardian issues emergency notification to all teams
