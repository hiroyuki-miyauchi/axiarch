## Part I. 資金調達の哲学

### 1.1. 資金調達の3つの誤り

- **Rule 15.001**: 以下の思考パターンを資金調達における危険なアンチパターンとして定義する

| アンチパターン | 問題点 |
|:------------|:------|
| **Fundraising as Validation** | 調達額を成功の証明と錯覚する。VC マネーは PMF の代替にならない |
| **Raise as Much as Possible** | 過剰な資本調達は希薄化の拡大と規律の喪失を招く |
| **Valuation Maximization** | 高すぎる評価額は次のラウンドおよびExitを困難にする（Down Round リスク） |

### 1.2. Default Alive（デフォルト生存）の原則

- **Rule 15.002**: 調達前に **Default Alive** の状態であることを目指す（Paul Graham）
  - Default Alive = 現在の成長率とコストのまま進んだ場合、自然黒字化する状態
  - Default Alive でない段階で調達する場合、そのリスクを明確に認識した上で行う

### 1.3. Ramen Profitability First

- **Rule 15.003**: 外部資本に依存しない最小生存状態（Ramen Profitability）を早期に達成することを優先する
  - 創業者の生活費が賄える最低限の収益状態
  - これにより調達交渉での「強い立場」を作る（Desperate な状態での交渉は常に不利）

## Part II. 資金調達ステージとタイミング

### 2.1. ステージ別の特徴と基準指標

- **Rule 15.010**: 調達ステージと対応する達成基準を以下のフレームで整理する

| ステージ | 調達額目安 | 達成基準（目安） | 主な投資家 |
|:--------|:---------|:------------|:--------|
| **Pre-seed** | 〜3,000万円 | アイデア + チーム | エンジェル / FFF |
| **Seed** | 3,000万〜1.5億円 | PMFの兆し + MRR 200万円 | Seed VC / エンジェル |
| **Series A** | 1.5億〜10億円 | PMF確認 + MRR 1,000万円以上 + 成長率 MoM 10%+ | A ラウンドVC |
| **Series B** | 10億〜50億円 | Unit Economicsの成立 + ARR 10億円前後 | GroVC / CVC |
| **Series C+** | 50億円〜 | スケール確認 + 市場リーダーシップ | PE / 大手VC |

### 2.2. 調達タイミングの法則

- **Rule 15.011**: 調達タイミングは以下の原則で判断する

```
タイミングの鉄則:
1. 必要になる12ヶ月前から動き始める（2〜3ヶ月かかる）
2. ランウェイが12ヶ月以上ある状態で調達を開始する
3. 強いモメンタム（成長数字）がある時に動く
4. Desperate な状態での交渉は条件が悪化する

「お金が余っている時に借り、足りない時に借りてはいけない」
```

### 2.3. エクイティ vs デット vs ブートストラップ

- **Rule 15.012**: 資金調達の形態を以下の基準で選択する

| 方法 | 適しているケース | 注意点 |
|:----|:------------|:------|
| **エクイティ** | 高成長の一気がけが必要 | 希薄化 / 株主への説明責任 |
| **デット（融資）** | キャッシュフローが安定している | 返済義務 / 担保要件 |
| **Revenue-based** | SaaS の安定MRRがある | 返済条件は要精査 |
| **ブートストラップ** | ニッチで高利益 / 長期所有にこだわる | 成長速度に限界 |

## Part III. ピッチ設計プロトコル

### 3.1. ピッチデックの構成（標準15スライド）

- **Rule 15.020**: ピッチデックは以下の構成を基本とする

```
必須スライド（10枚）:
1. Cover（会社名・タグライン・ロゴ）
2. Problem（課題の定義・規模）
3. Solution（プロダクトの解決策・デモ）
4. Market Size（TAM / SAM / SOM）
5. Business Model（収益の仕組み・価格）
6. Traction（実績・KPI・成長グラフ）
7. Go-to-Market（獲得戦略）
8. Competition（競合比較・ポジショニング）
9. Team（創業者・主要メンバーの背景）
10. Financials（3年予測・調達額・使途）

補足スライド（5枚 Appendix）:
11. Unit Economics（CAC / LTV / Magic Number）
12. Product Roadmap
13. Customer Testimonials / Case Study
14. Technology / Architecture
15. Cap Table（現在の持分構成）
```

### 3.2. Traction スライドの設計原則

- **Rule 15.021**: Traction スライドは最も重要なスライドである

```
良い Traction の示し方:
✅ 成長率を強調する（MoM 20% 成長 = 年率792%）
✅ ロゴを見せる（著名顧客・ブランド名の掲示）
✅ 定性 + 定量の組み合わせ（NPS + MRR 数字）
✅ ビフォー/アフターで顧客の成功を示す

避けること:
❌ 絶対値のみ（10万ユーザー？それがどれくらいすごいか不明）
❌ Vanity Metrics（PV数・登録数のみ。エンゲージメントを示す）
❌ 恣意的な期間選択（成長が良い期間だけを切り取る）
```

### 3.3. The Narrativeの設計

- **Rule 15.022**: ピッチの全体的な物語（Narrative）を1つの一貫したストーリーとして設計する

```
推奨 Narrative 構造:
1. 世界の変化（Why Now）: 技術・市場の大きな変化が起きている
2. 課題（Problem）: それによって生まれた/顕在化した問題
3. 解決（Solution）: 我々はその問題をこう解く
4. 証明（Traction）: 市場がそれを求めているという証拠
5. ビジョン（Vision）: 5年後にどんな世界を作るか
```

## Part IV. 投資家選定・アウトリーチ

### 4.1. 投資家リストの構築プロトコル

- **Rule 15.030**: 投資家アウトリーチは以下のファネル管理で行う

```
Target: 投資家候補リスト（200社以上）
  ↓ リサーチ
Qualified: 自社ステージ・テーマに合う（50社）
  ↓ Warm Intro の確認
Priority: 紹介ルートがある投資家（20社）
  ↓ ミーティング
Meeting: 初回ミーティング実施（10社）
  ↓ 興味度確認
Active: 深掘りが進む投資家（5社）
  ↓ Term Sheet
Closing: ターム提示（1-3社）
```

### 4.2. Warm Intro（紹介）の重要性

- **Rule 15.031**: Cold での投資家アウトリーチの返信率は1%以下である。**必ず紹介（Warm Intro）を通じてアプローチする**
  - 紹介元: ポートフォリオ創業者 > LP > 他のVC > アドバイザー > 業界知人
  - 直接アプローチは紹介が全く取れない場合の最終手段とする

### 4.3. 投資家フィット（Investor Fit）の評価

- **Rule 15.032**: 投資家を以下の軸で評価し、「最高の投資家」を選ぶ

| 評価軸 | 確認すべき内容 |
|:------|:------------|
| **Thesis Fit** | 投資テーマが自社ステージ・領域と一致するか |
| **Portfolio Value-add** | ポートフォリオに自社の潜在顧客・パートナーがいるか |
| **Founder Reputation** | ポートフォリオ創業者の評判（レファレンスチェック） |
| **Board Value** | 投資家がboard memとして何を提供できるか |
| **Co-investment** | 次のラウンドにも参加する意思・実績があるか |

## Part V. タームシートと条件交渉

### 5.1. タームシートの主要条件

- **Rule 15.040**: タームシートの以下の条件について、法務専門家と連携した上で判断する

| 条件 | 注意点 |
|:----|:------|
| **Valuation (Pre/Post-money)** | Option Pool Shuffle に注意（VC 有利な条件に惑わされない） |
| **Liquidation Preference** | 1x Non-participating が創業者フレンドリー。2x+ は警戒する |
| **Anti-dilution** | Full Ratchet は避ける。Broad-based Weighted Average が標準 |
| **Pro-rata Rights** | 投資家が次ラウンドで比率維持できる権利。認めてよい |
| **Board Composition** | 創業者が board majority を維持できる構成を維持する |
| **Information Rights** | 月次・四半期財務報告の義務範囲を確認する |

### 5.2. Red Flag 条件リスト

- **Rule 15.041**: 以下の条件が含まれる場合は要注意として法務に確認を求める

```
警戒すべき条件（Red Flags）:
❌ Participating Preferred Stock（清算優先権 + 残余参加権の二重取り）
❌ Drag-along Right なしで創業者保護が弱い
❌ 投資家のコントロールが過大: 雇用/解雇の同意権、重要決定の拒否権
❌ 過度な Ratchet 条項（次ラウンドのValuationに連動した希薄化）
❌ 広すぎる ROFR（先買権）：Secondaryでの売却を制限する
```

### 5.3. Competing Offers（競合オファー）戦略

- **Rule 15.042**: 複数の投資家から同時にタームを取得することが最も有利な交渉を生む
  - **FOMO 戦略**: 「Interest が高まっているため今月末に決定します」というDeadlineを設ける
  - **Competitive Tension**: 特定の投資家に Best の条件を「他の投資家も同条件なら」という形で競わせる

## Part VI. デューデリジェンス対応

### 6.1. Data Room の構成

- **Rule 15.050**: デューデリジェンス（DD）に備え、以下のドキュメントを常時 Data Room に整備する

```
Data Room 構成（推奨）:
├── Corporate/（法人設立・株主名簿・議事録）
├── Financials/（月次P&L・BS・CF・予算）
├── Product/（Product Roadmap・アーキテクチャ図）
├── Legal/（契約書・IP権利・訴訟リスク）
├── HR/（組織図・雇用契約・ストックオプション）
├── Customers/（顧客リスト・MRR明細・Churn分析）
├── Technology/（コード品質・セキュリティ）
└── Market/（市場調査・競合分析）
```

### 6.2. DD 対応の原則

- **Rule 15.051**: DD における情報開示の原則を以下の通り定義する
  - **Proactive Disclosure**: 問題は自発的に開示する。隠蔽が後で発覚した場合のダメージは開示より大きい
  - **Consistency**: 全ての投資家に同じ情報を提供し、情報の非対称を作らない
  - **Completeness**: 「知らなかった」は許されない。自社の状態を完全に把握した上で対応する

## Part VII. 投資家リレーションズ（IR）

### 7.1. 月次投資家レポートの標準フォーマット

- **Rule 15.060**: 投資家に毎月の月次レポートを送付することを義務とする

```markdown
# [月次] 投資家レポート - YYYY年MM月

## TL;DR（3行サマリー）
- WIN: 〇〇が達成 / 〇〇のCustomer獲得
- MISS: 〇〇が未達 / 理由と対策
- ASK: 今月お願いしたいこと

## Key Metrics
| 指標 | 今月 | 先月 | MoM |
| MRR | ¥〇〇M | ¥〇〇M | +〇% |
| 顧客数 | 〇社 | 〇社 | +〇% |

## Highlights（良かったこと）

## Lowlights（課題・学び）

## Help Needed（投資家へのお願い）
- 採用: ○○エンジニアを探しています
- 紹介: ○○業界の顧客企業を紹介ください
```

### 7.2. Board Meeting の運営プロトコル

- **Rule 15.061**: Board Meeting は以下のプロトコルで運営する

```
頻度: 月次（Series A前）/ 隔月（Series A以降）
Board Pack 事前配布: 会議72時間前
構成:
  1. 業績レビュー（KPI・財務）
  2. 主要課題のディスカッション
  3. 意思決定の必要事項
  4. Next Actions と担当

禁止: Board Meeting での「驚き」。重大な悪い情報は事前に連絡する
```

### 7.3. 投資家への Early Warning プロトコル

- **Rule 15.062**: 以下の重大イベントが発生した場合、24時間以内に全投資家に通知する

```
Early Warning が必要な事象:
- ランウェイが6ヶ月を切る見込みになった
- Key Person（創業者・CTO等）が離脱する
- 重大なセキュリティインシデント
- 主要顧客（MRRの10%以上）の解約
- 重要な法的リスクの発生
- 競合による重大な動き（買収・大型調達等）
```
