# 13. ストアガイドラインとコンプライアンス (Store Guidelines & Compliance)

## 1. Apple App Store (iOS)
*   **Human Interface Guidelines (HIG)**:
    *   **デザイン準拠**: AppleのHIGに違反するUI（システムと紛らわしいボタン、戻る操作の欠如）はリジェクト対象です。徹底的に準拠します。
*   **ビジネスモデルと課金**:
    *   **IAP必須**: デジタルコンテンツの販売には必ず **In-App Purchase (IAP)** を使用します。外部決済への誘導リンク（Webで購入など）は厳禁です（リーダーアプリ特例を除く）。
*   **プライバシー (Privacy & Data)**:
    *   **App Tracking Transparency (ATT)**: IDFA（広告識別子）を取得する場合は、必ずユーザーの許可を求めるポップアップを実装します。
    *   **プライバシーマニフェスト (Privacy Manifests) [2025必須]**:
        *   `PrivacyInfo.xcprivacy` ファイルを必ず含め、アプリおよびサードパーティSDKが収集するデータタイプと使用目的を正確に記述します。
        *   **API使用理由**: ディスク容量や起動時間などのAPIを使用する場合、その正当な理由（Required Reason API）を宣言する必要があります。
    *   **データ最小化**: 機能に必要のないデータは収集しません。

## 2. Google Play Store (Android)
*   **ターゲットAPIレベル**:
    *   **最新維持**: Googleは定期的にターゲットAPIレベルの引き上げを要求します。これに遅れるとアップデートができなくなるため、常に最新のAndroidバージョンに対応します。
*   **Play Integrity API**:
    *   **改ざん検知**: アプリが改ざんされていないか、正規のデバイスで実行されているかを検証するために **Play Integrity API** を導入します。
*   **通報とブロック**:
    *   **UGC要件**: ユーザー生成コンテンツ（UGC）を含むアプリは、不適切なコンテンツの報告・ブロック機能を実装し、24時間以内に対応する体制を整える必要があります（2025年新ポリシー）。

## 3. プラットフォームAPI規約 (Platform API Terms)
*   **Google Maps Platform**:
    *   **キャッシュ禁止**: 地図データの保存・キャッシュは利用規約で厳しく制限されています。原則として毎回APIから取得します。
    *   **帰属表示**: Googleロゴと著作権表示を隠したり改変したりすることは禁止です。
*   **YouTube Data API**:
    *   **バックグラウンド再生禁止**: APIを使用してバックグラウンド再生を実装することは規約違反です。
*   **Sign in with Apple**:
    *   **義務**: Googleログイン等のサードパーティ認証を実装する場合、**Sign in with Apple** も同等のプロミネンスで実装することがガイドラインで義務付けられています。

## 4. ASO (App Store Optimization)
*   **キーワード戦略 (Keyword Strategy)**:
    *   **メタデータ**: タイトル、サブタイトル、キーワードフィールドには、検索ボリュームが高く、かつ競合性が適切なキーワードを戦略的に配置します。
    *   **ローカライズ**: 単なる翻訳ではなく、各国の検索トレンドに合わせたキーワード選定を行います。
*   **クリエイティブ最適化 (Creative Optimization)**:
    *   **A/Bテスト**: スクリーンショット、アプリアイコン、プレビュー動画は、**Product Page Optimization (iOS)** および **Store Listing Experiments (Android)** を使用して常にA/Bテストを行い、コンバージョン率（CVR）を最大化します。
    *   **動画**: 最初の3秒でアプリの価値が伝わるプレビュー動画を必ず用意します。

## 4. 共通の禁止事項とチェックリスト (Common Rejections & Checklist)
*   **申請前チェックリスト (Pre-Submission Checklist)**:
    *   [ ] **未完成機能**: 「Coming Soon」やベータ版の表記、ダミーテキスト（Lorem Ipsum）は全て削除したか？
    *   [ ] **クラッシュ**: TestFlight/内部テストでクラッシュ率が0%であることを確認したか？
    *   [ ] **リンク切れ**: プライバシーポリシー、サポートURL、利用規約のリンクは有効か？
    *   [ ] **不適切なコンテンツ**: 暴力、ポルノ、差別的なコンテンツが含まれていないか？（UGCの場合はフィルタリング機能があるか？）
    *   [ ] **知的財産権**: アプリアイコンや名称が他社の商標を侵害していないか？
    *   [ ] **ログイン情報**: 審査員用のデモアカウント（ID/Password）を備考欄に記載したか？（2段階認証はOFFにするか、テスト用コードを提供する）
*   **メタデータ**:
    *   スクリーンショットや説明文に、実際のアプリと異なる機能や、誤解を招く表現（「世界一」「No.1」など根拠のない主張）を含めません。
