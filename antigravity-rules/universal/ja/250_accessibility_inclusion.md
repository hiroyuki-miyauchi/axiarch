# 250. アクセシビリティとインクルージョン (Accessibility & Inclusion)

## 1. WCAG 2.1 AA 準拠 (WCAG 2.1 AA Compliance)
*   **法的・倫理的基準**:
    *   Web Content Accessibility Guidelines (WCAG) 2.1 のレベルAAへの準拠を目指します。これは法的リスク回避だけでなく、全てのユーザーに価値を届けるための倫理的責務です。
*   **コントラスト比**:
    *   テキストと背景のコントラスト比は少なくとも **4.5:1** を確保します（大きな文字は3:1）。Figmaのプラグイン（Stark等）でデザイン段階からチェックします。

## 2. 支援技術への対応 (Assistive Technology)
*   **スクリーンリーダー**:
    *   **セマンティックHTML**: `div` ではなく `button`, `nav`, `main` などの適切なタグを使用します。
    *   **代替テキスト**: 画像には必ず `alt` 属性を付与します（装飾目的の場合は空文字 `alt=""`）。
    *   **ネイティブ対応**: iOSのVoiceOver、AndroidのTalkBackで主要な操作フローが完了できることをテストします。
*   **キーボード操作**:
    *   マウスやタッチパネルを使えないユーザーのために、全てのインタラクティブ要素がキーボード（Tabキー）だけで操作できるようにします。フォーカスインジケータ（青枠など）を消さないように注意します。

## 3. インクルーシブデザイン (Inclusive Design)
*   **ダークモード**:
    *   **Dark Mode First**: OSの設定に合わせて自動的に切り替わるダークモードを実装します。単なる色反転ではなく、目に優しい配色を設計します。
*   **Dynamic Type**:
    *   ユーザーがOS設定で文字サイズを大きくした場合、アプリ内の文字もそれに追従して拡大するように実装します（レイアウト崩れを防ぐ）。
*   **多様性への配慮**:
    *   性別選択などで「その他」や「回答しない」を含めるなど、多様なバックグラウンドを持つユーザーを排除しないUI/UXを心がけます。
